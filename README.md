### 操作与资源

- 物理区 = **扭蛋移动范围** 324×343 @ **(26,165)**。
- **扭蛋掉落成功判定范围**（Frame「扭蛋机_未抓起扭蛋」xACw5）：机台 **(31,444) 83×56**（play **5,279**，用于 `heldCenterInSuccessZone` / 挡板后进入判定等）。
- **扭蛋掉落范围**（填充图层 qIweh）：机台 **(31,184) 83×388**。从爪 `drop`（`+` 补蛋不算）：**点 drop 时蛋心已在成功判定区内** → 立即按竖槽下落并消失。**否则**全程 **Matter 真实碰撞**（墙、地、挡板等），需 **先与「挡板范围」发生过碰撞**，之后 **蛋心再进入成功判定区**，才切换为沿该槽竖直下落（`vx=0`、无碰撞）并在穿出底边（play **y = 19+388**）后移除。
- 描边版「扭蛋掉落范围」240×343 @ **(110,165)**（`DROP_ZONE`）：**`+` 每点一次依次补 4 颗**（间隔约 140ms），每颗生成 x 与横向反弹均限制在此矩形内随机；场上满 10 颗则不再生成。
- **限时挑战（点 + 后）**：顶部 **倒计时** 对齐 Frame「扭蛋机_未抓起扭蛋」文本 **5pGt2** 机台 **(115.5,51.5)**，默认 **10s**（`倒计时：HH:MM:SS`）。每次 **成功掉落计分**（蛋心进入「积分动画触发范围」）在 **倒计时进行中** 时 **+5s**，并播放 **`+5s.png`** 动效（位置同 Frame「扭蛋机_成功掉落」图层 **EhMck** 机台 **(142,7) 88×49**，效果同积分+1）。**倒计时归零** 时播放 **`明天再来吧.png`**（Frame「扭蛋机_失败」**1ENjH** 机台 **(75,316) 236×59**）；**归零后**再成功掉落 **不增加积分、不播积分+1**（直至再次点 **`+`** 开新轮）。从未点 **`+`** 时仍按原逻辑可正常计分。**RESET** 结束挑战并隐藏倒计时。
- **积分**：Frame「扭蛋机_未抓起扭蛋」group「积分」机台 **(72,585) 231×47**；`RESET` 清零。
- **积分动画触发范围**（Frame「扭蛋机_成功掉落」图层 lIauM）：机台 **(31,459) 85×113**（play **5,294**）。竖槽下落过程中 **蛋心进入此矩形** 时才 **积分 +1** 并播放「积分+1」动效，**每颗蛋最多触发一次**（未经过该区域即穿出底边则不得分）。
- **积分+1 动效**：Frame「扭蛋机_成功掉落」图层 t5jaF，机台 **(148,539) 78×72**，资源 `积分+1.png`，自下而上移动并淡入淡出。
